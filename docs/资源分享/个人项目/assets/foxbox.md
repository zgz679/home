## 1. 项目简介

**Foxbox** 是一个基于 **Go** 的轻量级 CLI 工具，旨在简化开发者在容器环境下运行 CLI 工具的流程。
它的设计目标是：

- **环境隔离** - 每次运行工具都在干净、可复现的容器环境中，避免本地环境污染。

- **简化操作** - 用户无需关注复杂的 Docker/Podman 命令，像本地运行命令一样方便。

- **自动化管理** - 自动处理目录挂载、工具版本管理等繁琐操作。

需要指出的是，Foxbox 只会调用底层容器工具完成任务，不会直接管理容器生命周期。

---

## 2. 技术栈

Foxbox 的技术栈如下

| 技术栈 | 具体选型 | 特点 |
| :----: | :------: | :--: |
| 开发语言 | Golang | 静态编译、跨平台、开发规范简洁 |
| 容器运行时 | Podman / Docker | 主流容器运行环境，同时支持 CLI / API 调用方式 |
| 配置文件 | YAML单文件 | 简洁、便于迁移维护，便于开发 |

此外，项目本身还包括了以下技术栈

- 基于 Makefile + UPX 的二进制构建

- 基于 GitHub Actions 和语义化版本号的 CI/CD

---


## 3. 项目价值

- 目标 - 降低容器化开发工具的使用门槛，让实验环境和开发环境可复现。

- **应用场景**：

    - 多版本、多语言编译器管理 （gcc、clang、go、npm 等）

    - 容器化、可移植、封装的开发环境管理 (支持自行构建的容器镜像)
